## Master Session — Software Development Lifecycle

Alongside the six courses, every cohort member attends one dedicated **Master Session** on the **Software Development Lifecycle (SDLC)**.

This session covers what professional teams actually do between "idea" and "shipped product" — the planning, development, testing, deployment, and maintenance phases that textbooks describe but most bootcamps skip entirely.

You will leave understanding:

- How a feature moves from a business requirement to a deployed release
- The difference between waterfall, agile, and continuous delivery workflows
- What happens in each phase of development and who is responsible
- How to read a sprint board, write a ticket, and participate in a code review
- Why testing and monitoring are not optional extras

**This session applies directly to every role** — whether you are joining a product team as a junior developer, freelancing, or building your own product. Knowing how software is built at a team level is what makes you hireable on day one.

## What Makes This Program Different

**1. Fundamentals first, always.**
Every course in the Web Fundamentals track exists because skipping it causes real pain later. You will understand HTML semantics before you write a single line of CSS. You will understand the DOM before you touch React. This is intentional.

**2. No framework dependency fog.**
You will know exactly why React exists — because you will have built things without it first and felt the friction it solves. This makes you a stronger developer in any framework.

**3. AI is not a gimmick here.**
Course 6 is not "how to use ChatGPT." It is a technical course on integrating AI capabilities into production React applications. You will write real API calls, build real streaming interfaces, and deploy real features. This is the skill that the next generation of frontend developers will need.

**4. Git from day one.**
Course 4 runs alongside all other courses. You will commit your first HTML file and never look back. By the time you graduate, your GitHub profile tells a story.

**5. The master session closes the gap.**
Understanding how software is built professionally — the SDLC — is the difference between a developer who can execute tasks and one who can participate fully in a product team. Most programs never cover this. This one makes it a dedicated session.
